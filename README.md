**Map 구조를 선택한 이유**  
- List는 특정 날짜를 찾으려면 전체를 순회해야함
- 같은 날짜에 여러 개 저장되는 문제 → 그날 하루 기분이라는 취지와 맞지 않다 생각함
- Map은 날짜에 바로 접근할 수 있음
- 날짜당 하나만 저장되는 구조기 떄문에 달력 앱에 적합하다 판단
- 파이썬에서 배웠던 딕셔너리(dict)와 동일한 개념임

**DrawBehind** 
UI 요소 뒤에 배경을 직접 그릴수 있는 기능  
보통 compose에선 Box나 Text 같은 요소는 정해진 모양만 가질수 있지만  
drawbehind를 사용하면 그 요소 뒤에 원하는 배경을 그릴 수 있기 때문에 사용함  
기본 이모지는 너무 식상해서 사용하게됨
